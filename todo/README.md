## Task Format

```markdown
### Task Title
- **Status**: Pending | In Progress | Blocked | Done
- **Priority**: P0 | P1 | P2 | P3
- **Assigned**: @username (optional)
- **Created**: YYYY-MM-DD
- **Due**: YYYY-MM-DD (optional)
- **Description**: What needs to be done
- **Notes**: Additional context or updates
- **Blockers**: What's preventing progress (if blocked)
```

## Status Definitions

| Status | Description |
|--------|-------------|
| Pending | Not yet started |
| In Progress | Currently being worked on |
| Blocked | Waiting on something |
| Done | Completed |

## Priority Levels

| Priority | Description |
|----------|-------------|
| P0 | Critical / Urgent |
| P1 | High priority |
| P2 | Normal priority |
| P3 | Low priority |

## Workflow

1. **Add**: Create new tasks in the appropriate file
2. **Assign**: Self-assign or coordinate with team
3. **Update**: Keep status current
4. **Complete**: Mark done when finished
5. **Archive**: Move to done section or delete

## Quick Add

For quick task capture, use this minimal format:

```markdown
- [ ] Task description (P2)
```

Then expand with full details when ready to work on it.
